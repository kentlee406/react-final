以下這邊是助教針對程式碼的修改建議。

React

後台可新增 layout 或路由保護，統一在這檢查登入狀態，就不用在商品、訂單頁各寫一次
普通按鈕建議都要加上 type="button"，確保在不同瀏覽器都能如預期運作（購物車、後台頁面）
結帳表單的普通欄位建議補上 type="text"，並設定對應的 label for 與 input id


Bootstrap

格線系統 .col-**-* 同層的 .col-12 可以移除
SCSS 檔案可以參考 Sass 7-1 Pattern 進行分類，未來會較好維護


網站設定

可多加美化 index.html 中的 meta，例如：新增社群資訊設定（open graph Metadata）。另外請多加注意 lang 要調整至相關語系，例如調整成 「zh-Hant-TW」，若是英語系網站則可以保留預設 「en」


網站操作

跳轉頁面後可以回到頁面最上方，尤其是商品切換分頁的部分
前台｜

購物車品項調整數量的部分，當選擇較多的數量時會需要發出多次請求，可考慮使用 debounce、確認按鈕或其他方式優化
刪除購物車項目也可以回饋使用者操作成功
價格顯示的部分可以加上千分符（例如：2400 → 2,400）會更清楚
後台｜

後台編輯商品資料若不需要儲存，關閉再開啟後可以回復到原本的資料狀態（目前編輯後若沒有儲存重啟 Modal 後會是先前未儲存的資料狀態）


內容豐富度

首頁可以再試著加上幾個區塊或增加內容會更豐富
結帳流程可考慮拆分成幾頁，例如：確認購物車 → 填寫資料 → 已完成購買
